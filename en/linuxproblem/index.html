二、Common Errors

1. Issues potentially caused by graphics drivers:

If any of the following issues occur, configure the graphics driver first:

  • Screen flickering
  • Dual-monitor resolution issues

2. Unable to adjust screen brightness

Use xrandr to force adjust screen brightness:

1
2
3
4
5

# 1. Identify your monitor ID
xrandr | grep " connected" # Assuming the output starts with DP-4

# 2. Adjust brightness to 0.6 (range 0.0-1.0)
xrandr --output DP-4 --brightness 0.6

3. WiFi icon disappeared

The WiFi icon disappears because Ubuntu, while executing sudo apt update && sudo apt upgrade, enables the automatic kernel upgrade service (unattended-upgrades) by default. This causes the kernel to upgrade from 5.15.0-67-generic to 5.15.0-130-generic. Since the new kernel is incompatible with the AX211 network card driver, the driver fails to load, firmware files cannot be loaded correctly, and the kernel module versions do not match, ultimately causing the wireless card to malfunction and the WiFi icon to vanish.

You can use the uname -r command to check the current kernel version. If it differs from the version when the WiFi was working, it is likely that the automatic system update caused a kernel upgrade that is incompatible with the AX211 driver.

  1. Roll back to the previous kernel version
    After rebooting, select “Advanced options for Ubuntu” at the boot interface to enter the GRUB menu. Then, select the original kernel version (5.15.0-67-generic) to boot and test if the WiFi functionality is restored.

  2. Lock the current kernel version

1
2
3
4

sudo apt-mark hold linux-image-5.15.0-67-generic
sudo apt-mark hold linux-headers-5.15.0-67-generic
sudo apt-mark hold linux-modules-5.15.0-67-generic
sudo apt-mark hold linux-modules-extra-5.15.0-67-generic
  1. Verification Enterapt-mark showholdto check if the lock was successful
  2. Modify GRUB configuration sudo nano /etc/default/grub

4. Double-click icon to minimize

Configure: gsettings set org.gnome.shell.extensions.dash-to-dock click-action 'minimize' Restore: gsettings reset org.gnome.shell.extensions.dash-to-dock click-action
