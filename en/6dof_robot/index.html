1. Project Overview

This project designs a 6-DOF (Degree of Freedom) fully-actuated underwater robot (AUV) for underwater grasping tasks and high-performance motion control. Compared with traditional under-actuated or over-actuated AUVs, the 6-DOF fully-actuated scheme can achieve higher-level motion control capability on a compact design basis. Additionally, to simultaneously satisfy high-precision underwater grasping and high-performance motion control, this paper designs two motion modes for the AUV: grasping mode and cruising mode. The grasping mode uses traditional control schemes to improve precision, while the cruising mode uses Deep Reinforcement Learning (DRL) for smooth motion control. Furthermore, the AUV is designed with 4 expansion module interfaces for connecting various extension modules to meet diverse underwater operation tasks.

1.1 Design Objectives

  • High-performance mechanical structure design
  • Traditional control scheme design
  • DRL control scheme design
  • Autonomous path navigation
  • Target recognition and grasping
  • Expansion module development

1.2 Technical Support

2. Structure Design

2.1 Thruster Distribution Scheme

Thruster distribution is divided into three categories: under-actuated, fully-actuated, and over-actuated.

TypeCharacteristicsApplication
Under-actuatedFewer thrusters than DOFsSimple observation tasks
Fully-actuatedEqual thrusters and DOFsPrecise operation tasks
Over-actuatedMore thrusters than DOFsHigh redundancy tasks

Considering that the robot needs to achieve high-level grasping tasks on a compact design basis, this design adopts the 6-DOF fully-actuated scheme.

The conventional fully-actuated scheme uses 6 fixed independent thrusters with mature control strategies, but is not conducive to compact design. Therefore, this paper designs a 4-thruster + 2-servo asymmetric vector propulsion scheme:

  • 4 main thrusters control thrust magnitude
  • 2 servos control thrust direction of the 4 thrusters
  • 4 thrusters are arranged in a cross pattern; upper and lower thrusters rotate in opposite directions, left and right thrusters rotate in the same direction

3. Control Scheme

3.1 PID-based Control Scheme

For grasping mode, to ensure motion precision, a PID controller is used for low-level motion control:

class PIDController {
public:
    double kp, ki, kd;
    double setpoint, current_value;
    double integral = 0;
    double last_error = 0;
    
    double compute(double feedback) {
        double error = setpoint - feedback;
        integral += error;
        double derivative = error - last_error;
        last_error = error;
        return kp * error + ki * integral + kd * derivative;
    }
};

3.3.1 Scene Configuration (Scene)

  • Number of Environments: Supports large-scale parallel training
  • Terrain Type: Flat plane (plane) underwater environment
  • Robot: AUV Unmanned Underwater Vehicle

3.3.2 Action Space (Actions)

The AUV has two types of actuators:

Actuator TypeJoint NameFunctionAction Scale
Servojoint_servo_up, joint_servo_leftControl rudder angle±1.5 rad
Thrusterjoint_prop_up, joint_prop_down, joint_prop_left, joint_prop_rightThrust magnitude(-30N, 43N)

3.3.3 Command Space (Commands)

The velocity command generator uses UniformVelocityCommandCfg, generating commands including:

  • Linear Velocity Command:
    • X-axis (forward/backward): -0.5 ~ 2.0 m/s
  • Angular Velocity Command:
    • X-axis (roll): -1.0 ~ 1.0 rad/s
    • Y-axis (pitch): -1.0 ~ 1.0 rad/s
    • Z-axis (yaw): -1.0 ~ 1.0 rad/s

3.3.5 Reward Function (Rewards)

Reward design is divided into core objective rewards and penalty terms:

Core Rewards:

  • track_linear_velocity: Track target linear velocity
  • track_angular_velocity: Track target angular velocity
  • alive: Survival reward

Penalty Terms:

  • action_rate_l2: Penalize action change rate to prevent motor jitter (weight -0.01)
  • action_l2: Penalize action magnitude to encourage energy saving (weight -0.005)
  • joint_acc_l2: Penalize joint acceleration

3.3.6 Event System (Events)

  • Joint Reset: Reset robot joint state at the start of training
  • Hydrodynamics: Add resistance and buoyancy in water

3.3.7 Termination Conditions (Terminations)

  • time_out: Exceeds certain time
  • excessive_lin_vel: AUV linear velocity too fast termination
  • excessive_ang_vel: AUV angular velocity too fast termination

3.3.8 Curriculum Learning (Curriculum)

To improve training convergence, this paper sets up curriculum learning, dividing AUV training into 3 stages:

  • Stage 1: Train only one dimension of velocity or angular velocity
  • Stage 2: Train velocity and single-direction angular velocity
  • Stage 3: Train velocity and any-direction angular velocity
