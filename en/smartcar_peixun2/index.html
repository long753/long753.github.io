1.打开小车

首先检查电池的电压是否是在11.6~12.6V之间，不要过充过放；然后，将状态良好的电池插入小车之中并打开电源开关；最后，打开电源等待edgeboard亮绿灯。

2.连接电脑

2.1 有线连接（第一次连接）

  1. 用网线连接电脑和智能小车
  2. 打开终端，输入指令：（有可能连接不上，多试几次）
1

ssh edgeboard@paddlepi

输入密码：edgeboard，即可连接成功。 注： 如果这一步都连不上，多尝试几次，如果一直不行，找我换板卡

2.2 无线连接（最常用）

无线连接是paddle和你所使用的电脑都连接同一个热点的情况下常用的一种连接方式。第一次无线连接时，在终端中输入：

1

sudo nmtui

点击active connect连接你的热点即可。之后连接小车只需要你打开热点，等待小车连接上你的电脑后，直接在终端中输入：

1

ssh edgeboard@paddlepi

密码：edgeboard

3.通过ssh连接vscode

3.1安装插件

首先在vscode里面安装Remote - SSH，这一步很ez

3.2 在插件里面进入edgeboard

  1. 点击ssh-remote的“+”号
  2. 在输入栏中输入：
1
2

ssh edgeboard@paddlepi
密码：edgeboard
  1. 一直允许，配置文件点第一个就可以了（记住这个配置文件的位置）
  2. 在vscode里面打开car-work文件夹
1

/home/edgeboard/car-work

这一步，90%会失败。由于edgeboard太古老了，用的还是Ubuntu18.04的系统，因此需要大家将VScode的版本降低一下，再重新连接一下板卡。经过测试，1.85.2是最新的可以连接ubuntu18.04的版本，这是下载连接：https://code.visualstudio.com/updates/v1_85

3.3 配置密钥（可选）

如果你懒得每次用ssh连接板卡的时候都要输入密码，可以配置一下密钥，这样就不用每次都输入密码了。具体步骤如下：

  1. 在windows终端内输入指令： ssh-keygen -t ed25519 -C "your_email@example.com" 注意，更改一下your_email@example.com的内容，尽量改成你熟悉的，我个人比较喜欢用long753-win这种命名
  2. 将你生成的密钥对中的公钥（有.pub后缀）的内容复制下来，粘贴到edgeboard的.ssh/authorized_keys中
  3. 在windows中的./ssh/config里面输入，跟我视频里面的那个格式一样 IdentityFile C:\Users\cl284\.ssh/id_ed25519

4.打开VNC图形化界面

如果你还没有安装VNC，请打开网址https://www.realvnc.com/en/connect/download/viewer/，安装windows版本的VNC客户端，并按照安装提示安装好。

进入edgeboard终端后，输入指令：

1

vncserver :1 -localhost no

其中：:1表示图形化界面端口号，可以自行设置。优先使用1号端口，当端口被占用时，可以尝试其他端口,只使用1~4号端口。

当出现下面的提示则说明启动成功：

1
2
3
4
5
6

New 'paddlepi:1 (edgeboard)' desktop at :1 on machine paddlepi

Starting applications specified in /etc/X11/Xvnc-session
Log file is /home/edgeboard/.vnc/paddlepi:1.log

Use xtigervncviewer -SecurityTypes VncAuth,TLSVnc -passwd /home/edgeboard/.vnc/passwd paddlepi:1 to connect to the VNC server.

然后打开VNC,建立新的连接：（如果连接过，直接点你原来建立的就ok）

1
2

vncserver：paddlepi:5901
name: edgeboard

同时，如果使用的不是1端口，例如是2端口，对应的用户名应该为： vncserver：paddlepi:5901

#修复环境
sudo apt --fix-broken install -y
sudo apt update
sudo apt install -y build-essential xz-utils bison flex wget

#拉取源码
cd /home/edgeboard/Download
wget https://mirrors.nju.edu.cn/gnu/gcc/gcc-11.2.0/gcc-11.2.0.tar.xz
sha256sum gcc-11.2.0.tar.xz
tar -xf gcc-11.2.0.tar.xz

#拉取依赖并编译
cd gcc-11.2.0
./contrib/download_prerequisites
cd ..
mkdir -p gcc-11.2.0-build && cd gcc-11.2.0-build
../gcc-11.2.0/configure \
  --prefix=/opt/gcc-11.2 \
  --enable-languages=c,c++ \
  --disable-multilib \
  --enable-bootstrap
make -j"$(nproc)"  #需要很长一段时间，大概一坤时，请准备预留好时间，后续将很快
sudo make install

#修改PATH
echo 'export PATH=/opt/gcc-11.2/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/opt/gcc-11.2/lib64:${LD_LIBRARY_PATH}' >> ~/.bashrc
source ~/.bashrc

#验证
which gcc
gcc --version
g++ --version

5.2 libavdevice-dev 

1

sudo apt-get install libavdevice-dev

6.编译文件

在VNC界面中，打开终端，切换到工程目录，输入指令：

1
2
3
4
5
6

 cd /home/edgeboard/car-work  #切换到自己的那个工作空间里面
 rm -rf build
 mkdir build
 cd build
 cmake ..   #如果你修改了整个代码框架，可能需要修改一下CMakeList文件，如何修改可以去问AI
 make icar -j3

完全配置完全之后，保存你修改的代码后，进入build目录输入指令即可完成编译：

1

 make icar -j3

其中，“icar”表示编译的目标文件，“-j3”表示使用三个线程编译，可以加快编译时间。这里的编译时间会根据你的修改程度而变化。

7.运行程序

在终端中，切换到工程目录，输入指令：

1

 ./icar

此时小车便成功启动，结束小车的运动只需在终端中输入指令Ctrl+C即可。

问题汇总：

一、登录VNC时，需要root的密码，可是输入edgeboard也没用。 

1
2
3

cd /root/.vnc
touch xstartup
vim xtartup

在里面复制：

 2

unset SESSION_MANAGER        #将此行的注释去掉
exec /usr/bin/startxfce4        #将此行的注释去掉
#exec /etc/X11/xinit/xinitrc        #将此行的注释去掉

[ -x /etc/vnc/xstartup ] && exec /etc/vnc/xstartup
[ -r $HOME/.Xresources ] && xrdb $HOME/.Xresources
xsetroot -solid grey
vncconfig -iconic &
xterm -geometry 80×24+10+10 -ls -title “$VNCDESKTOP Desktop” &
gnome-session gnome           #添加这一句是连接时使用gnome 桌面环境
twm &

重新启动vnc：

1
2

vncserver -kill :1
sudo vncserver :1 -localhost no

二、edgeboard环境已经坏掉了，需要刷系统
