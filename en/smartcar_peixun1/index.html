This document covers only the fundamental aspects of Git and Linux systems, which should be sufficient for most scenarios encountered in the Smart Car Competition. Beyond these basics, there are many advanced topics not covered here. Based on my experience, these advanced features are less frequently used in the competition, so I recommend learning them only when you encounter a specific need.

Furthermore, there is no need to study Git and Linux “systematically” from scratch; doing so will consume a significant amount of your time with minimal return for the competition itself. I encourage everyone to learn through practice and application rather than blindly pursuing theoretical mastery. Given my own limitations, please be understanding if there are any errors or inaccuracies in this guide!

1. Git Essentials

1.1 Why Use Git?

Git is a widely-used distributed version control system, typically used in conjunction with GitHub or Gitee (Windows users can download it directly from the official Git website).

  • Remote Repositories: Create and manage codebases with GitHub/Gitee (Crucial).
  • Version Control: Track and manage different versions of your code.
  • Collaboration: Enable multiple team members to work on the same project.

Using GitHub or Gitee to create a repository and regularly back up your code is vital! This prevents code loss in case the car’s hardware fails unexpectedly. You should commit your code every time you finish a tuning session.

Recommended Workflow: Create local repositories on both the car’s onboard computer and team members’ laptops, all linked to a single remote repository. Push code from the car and Pull code to the laptops to avoid merge conflicts.

1.2 GitHub or Gitee?

Since our embedded boards (such as EdgeBoard or RK3588) usually connect via mobile hotspots, accessing GitHub directly can be unreliable without a proxy. For simplicity, I recommend using Gitee for code backups. The operations for Gitee and GitHub are nearly identical, so the skills learned here apply to both. Advanced users can still use GitHub via proxies or by setting up mirror repositories between the two platforms.

1.3 Creating a Remote Repository on Gitee

  1. Log in to Gitee.
  2. Click Create Repository, set a name, and set it to Private.
  3. Go to Management -> Repository Member Management -> Administrator -> Invite User. Add your teammates’ accounts and set their permissions to Manager.
  4. Now, you and your teammates can collaborate on this remote repository!

1.4 Creating a Local Repository

Create an empty folder in a directory (avoid Chinese characters in the path), open your terminal in that folder, and run:

1

git init

You now have an empty local Git repository.

1.5 Linking Local and Remote Repositories 

1

git remote add origin [https://gitee.com/yourname/yourworkSpace.git](https://gitee.com/yourname/yourworkSpace.git)

Verify the link using git: remote -v

1.6 Common Git Commands ⭐⭐⭐⭐⭐

To submit your first piece of code:

1
2
3
4

touch README.md
git add .
git commit -m "Your version info/message"
git push origin master

To pull code:

1
2

git clone [https://gitee.com/yourname/yourworkSpace.git](https://gitee.com/yourname/yourworkSpace.git)
git pull origin master

Force Push (Use only if you are certain your local version is the absolute latest!):git push -f origin master

Check current file status: git status

1.6.2 Code Version Control

View commit history: git log --oneline

Revert to a specific version: git reset (hash value)

1.6.3 Remote Repository Management

View the current remote repository: git remote -v

Add a remote repository: git remote add origin https://gitee.com/xxx/xxx.git

1.6.4 Branch Management

Branch management is a very important feature of Git, but I don’t recommend it for beginners. Firstly, branch management involves a significant amount of Git knowledge, and mastering git branch requires further study. Secondly, the intelligent vehicle competition project isn’t large, and collaboration among team members is relatively easy; there’s no need to overcomplicate things. However, I still suggest learning git branch, as it’s quite commonly used in large projects.

View branches: git branch Create branch: git branch branch_name Switch branch: git switch branch_name Merge branch: git merge branch_name Delete branch: git branch -d branch_name

2. Linux Basics

2.1 Linux in a Smart Car?

The default system for the board is Linux. We need to be proficient in using various Linux commands for file management, dependency acquisition, system configuration, etc. Here we will introduce some commonly used Linux commands during vehicle setup. Of course, the content of the Linux system is far more extensive than this; students can learn more during practical use.

Unlike Windows, most Linux operations are performed using the terminal. Therefore, we need to learn some commonly used bash commands.

2.2 Changing Directories

Enter a specified directory: cd /path/dir

Return to the user’s home directory: cd ~

Go back to the previous directory: cd ..

2.3 File Management

List all files in the current directory: ll, ls

Create a file: touch file.txt

Create a folder: mkdir dir

Delete a file: rm file.txt

Delete a folder: rm -rf dir/

Copy a file (source and index): cp a.txt b.txt

Copy a directory (source and index): cp -r dir1 dir2

Rename (source and index): mv a.txt b.txt

Move: mv a.txt dir

2.4 Vim Usage

Vim is a powerful tool that allows you to edit code directly in the terminal. Although code can be edited in VS Code, it’s still advisable to learn how to use Vim just in case.

Vim has three modes: Normal, Insert, and Command

  • Normal mode: This is the default mode when opening a file with Vim. In this mode, you can only view the code and use keyboard shortcuts to manipulate it.

  • Insert mode: Use i to switch from Normal mode to Insert mode, and use ESC to switch from Insert mode to Normal mode. In this mode, we can modify code.

  • Command mode: Enter : to enter, then enter certain commands to save and exit the file.

Related commands:

  • Modify file: vim file.txt

  • Save and exit: :wq

  • Force exit without saving: :q!

  • Force save and exit: :wq!

  • Learn other commands as needed.

2.5 Permission Management*

This section is only used when encountering a few unusual issues.

View permissions: ll

Modify permissions: chmod a+x file.txt (where r is read permission, w is write permission, and x is execute permission)

2.6 Other

Connect to hotspot: sudo nmtui
