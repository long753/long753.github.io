该文档只讲解了Git和Linux系统的基础内容，这部分内容可以应对智能车比赛的大多数情况。但是除了所讲解的基础内容，还有很多重要的内容我没有讲解。根据我的经验，这些内容在智能车比赛中使用的较少，所以请各位同学遇到的时候再进一步学习。此外，大家没有必要去系统的学习git和linux，这会消耗大家大量时间，而且对智能车比赛的帮助是很小的，因此希望大家可以在比赛中学习，应用中学习，而不是盲目追求系统学习。由于本人水平有限，若有讲解错误或者不恰当的地方请多多包含！

1. Git基础内容

1.1 Git有何用？

git是一个常用的分布式版本控制系统，跟github或者gitee配合使用（windows直接在git官网下载git就可以了）。

  • 跟GitHub和gitee配合，创建代码仓库（important）
  • 可以对代码进行版本管理
  • 可以多人协作 使用github或者gitee创建一个代码仓库，并且定期保存小车上的代码是十分重要的！！！以防小车突然暴毙之后，之前写的代码没了！！！每次调完车都要提交当前代码。 推荐的代码提交方案：在小车以及队员电脑上创建本地仓库，共同关联一个远程仓库，小车上push代码，队员电脑上pull代码，这样可以防止push冲突。

1.2 GitHub or gitee？

由于板卡（edgerboard或者rk3588）是连接的我们手机的热点，大概率是不能直接访问GitHub的，需要使用代理。因此，为了简单操作，推荐直接使用gitee来保存代码就可以了。此外，gitee和GitHub操作几乎没什么区别，因此我们培训的东西也是可以用于GitHub的。当然，如果你是GitHub的忠实用户，你可以在板卡上面使用代理或者在gitee和GitHub之间创建镜像仓库，这里不再赘述。

1.3 使用gitee创建远程仓库

这里我们将介绍如何使用gitee来创建一个远程仓库

  1. 登录gitee：https://gitee.com/
  2. 点击创建仓库，设置仓库名称，并将仓库改为私有。
  3. 点击管理->仓库成员管理->管理员->邀请用户->直接添加->仓库权限选为管理者->输入你的好队友的账户->添加
  4. 现在你就可以和你的好队友一起使用这个远程仓库了！

1.4 创建本地仓库

在非中文目录下创建一个空文件夹（比如桌面），然后在终端中进入该文件夹，最后输入指令：

1

git init

这样你就有一个空的本地仓库了

1.5 关联本地仓库和远程仓库 

1

git remote add origin https://gitee.com/yourname/yourworkSpace.git

使用git remote -v检查；

1.6 git的常用指令⭐⭐⭐⭐⭐

1.6.1提交、拉取代码（这一部分用的最多）

提交我们的第一个代码！

1
2
3
4

touch README.md
git add .
git commit -m"版本信息"
git push origin master

拉取一个代码

1
2

git clone https://gitee.com/yourname/yourworkSpace.git
git pull origin master

强制提交(一定要确保这个是最新的！！！！)

1

git push -f origin master

查看当前的文件状态：git status

1.6.2 代码版本管理

查看提交历史：git log --oneline 退回指定版本：git reset (哈希值)

1.6.3 远程仓库管理

查看当前远程仓库：git remote -v 添加远程仓库：git remote add origin https://gitee.com/xxx/xxx.git

1.6.4 分支管理

分支管理是git的一个很重要的功能，但是我不推荐初学者使用。第一个原因是分支管理这一部分涉及到较多的git知识，大家想要用好git branch需要进一步学习；第二个原因是智能车比赛这个项目不算大，各个同学的协作还是比较方便，没必要杀鸡用牛刀。但是还是建议各位学习一下git branch，这个东西在大型项目中还是比较常用的。 查看分支：git branch 创建分支：git branch branch_name 切换分支：git switch branch_name 合并分支：git merge branch_name 删除分支：git branch -d branch_name`

2. Linux基础内容

2.1 智能车的Linux？

板卡的默认系统是Linux，我们需要熟练使用Linux的各种指令来进行文件管理、依赖获取、系统配置等等。这里我们会介绍一些在调车过程中，常用的Linux指令，当然Linux系统的内容远远不止这些，剩下的东西同学们在实际使用过程中可以进一步学习。 Linux系统和Windows系统不同，Linux的大多数操作都是使用终端来实现的，因此我们需要学习一些常用的bash指令。

2.2 切换目录

进入指定目录：cd /path/dir 回到用户主目录：cd ~ 回到上一级：cd ..

2.3 文件管理

列出当前目录下的所有文件：ll ,ls

创建文件：touch file.txt 创建文件夹：mkdir dir 删除文件：rm file.txt 删除文件夹：rm -rf dir/

复制文件(前源后标)：cp a.txt b.txt 复制目录(前源后标)：cp -r dir1 dir2 重命名(前源后标)：mv a.txt b.txt 移动：mv a.txt dir

2.4 vim使用

vim是一个强大的可以直接在终端上修改代码的工具，虽然代码可以在vscode上面改，但是以防万一，各位还是学一下vim的使用 vim中有三种模式：Normal, Insert, Command

  • Normal模式：我们用vim打开这个文件的默认模式，这个模式下我们只能看代码，用快捷键对代码进行操作
  • Insert模式：使用i从Normal模式切换为Insert模式，使用ESC从Insert模式切换为Normal模式。在该模式下，我们可以修改代码
  • Command模式：输入:后进入，然后再输入一定指令即可实现文件的保存和退出

相关指令：

  • 修改文件：vim file.txt
  • 保存并退出：:wq
  • 强制不保存并退出：:q!
  • 强制保存退出：:wq!
  • 其余指令请在需要使用时学习

2.5 权限管理*

这一部分的内容只有在遇到少许猎奇问题的时候才会使用 查看权限：ll 修改权限：chmod a+x file.txt(其中，r是可读权限，w是可写权限，x是可执行权限)

2.6 其他

连接热点：sudo nmtui
