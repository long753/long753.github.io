1. 项目简介

第二十届全国大学生智能汽车竞赛（完全模型组）要求参赛队伍设计制作能够自主完成赛道识别、元素处理和路径规划任务的智能小车。赛道元素包含：直道、弯道、十字、圆环。AI元素包含：充电区、临时停车区、餐饮区、路障。该项目获得本次比赛的全国一等奖！

本项目基于EdgeBoard嵌入式平台，采用YOLOv3目标检测算法，结合传统图像处理方法，实现了完整的智能车控制系统。完整代码见：https://github.com/long753/SmartCar

2. 硬件平台

2.1 主控平台

本项目采用百度EdgeBoard作为主控平台，其主要参数如下：

参数规格
CPUARM Cortex-A53/A72
GPUMali-T860
算力3.0 TOPS
内存4GB LPDDR4
系统Ubuntu 18.04

EdgeBoard内置了PaddlePaddle推理引擎，支持多种深度学习模型的部署，同时具备较强的实时图像处理能力，非常适合智能车竞赛的应用场景。

2.2 传感器配置

  • 摄像头：无畸变110度，300帧工业相机（杰瑞微通）
  • 串口：USB转TTL串口，连接主控与edgeboard（上位机）
  • 电池：12V锂电池，为全车提供动力
  • TC264：主控芯片
  • IMU：提供小车的姿态信息

4. 关键技术实现

4.1 目标检测

采用YOLOv3-Mobilenet模型进行赛道元素检测，支持以下元素的识别：

  • 充电区标志 (BATTERY)
  • 黑盒子路障 (BLOCK)
  • 坡道 (BRIDGE)
  • 汉堡障碍物 (BURGER)
  • 锥桶 (CONE)
  • 斑马线 (CROSSWALK)
  • 行人 (PEDESTRIAN)
  • 虚线 (LINE)

void SmartCar::ai_process_mul() {
  auto ai_process_task{
    [](std::shared_ptr<Detection> detection){
      cv::Mat m;
      while(smartcar.ai_runable){
        if(!smartcar.ai_picture_update.load()) continue;
        m = std::move(smartcar.image_for_predict);
        detection->inference(m);  // 模型推理
        smartcar.predict_result = detection->results;
        smartcar.ai_picture_update = false;
      }
    }
  };
  predict_thread = std::jthread(ai_process_task, this->detection);
}

4.2 赛道边线提取(Findline.cpp)

功能描述

赛道边线提取是智能车视觉导航的核心功能，负责从二值化图像中精确提取左右赛道边界，为运动控制提供准确的路径信息。该系统基于八邻域算法实现自适应路径追踪，具有高精度和强鲁棒性的特点。

技术实现

1. 八邻域搜线算法

算法采用改进的八邻域算法，定义了两个方向的邻域模板进行赛道边缘追踪：

function search_line(binary_image):
    processed_image = morphologyEx(binary_image, MORPH_OPEN)  // 开运算去噪
    left_start = find_start_point(processed_image, 'left')
    right_start = find_start_point(processed_image, 'right')
    
    while not (left_stop and right_stop):
        next_left = find_next_point(left_current, seeds_l, processed_image)
        next_right = find_next_point(right_current, seeds_r, processed_image)
        
        if point_stuck(left_current): jump_to_new_point(left_current, processed_image)
        if point_stuck(right_current): jump_to_new_point(right_current, processed_image)
        
        left_current = next_left
        right_current = next_right
        left_points.add(left_current)
        right_points.add(right_current)
    
    calculate_edge_points()
    determine_line_type()

    if ringType == RingRight:

void Cross::Cross_repair_right(Findline &findline) {
    while(!findline.pointsEdgeLeft.empty() && findline.pointsEdgeLeft.back().col >10) {
        findline.pointsEdgeRight.emplace_back(findline.pointsEdgeLeft.back());
        findline.pointsEdgeLeft.pop_back();

    if(cross_back != cv::Point{0,0} && cross_down_point != cv::Point{0,0}) {
        for rowCur from down_point to back_point:
            colInterp = colStart + t * (colEnd - colStart)
            findline.pointsEdgeRight.add(mpoint(rowCur, colInterp))
    }
}

其他元素处理

系统还支持多种AI元素的处理：

  • 充电区 (ChargeDetection.cpp)：识别电池标志并执行充电流程

每个元素都采用专门的状态机和处理逻辑，确保在各种赛道环境下都能稳定运行。

void SmartCar::motion_control() {
  // 动态前瞻调整
  if(CarParams->qianzhan_change && control.motorSpeed <= CarParams->speedHigh && control.motorSpeed >= CarParams->speedLow){
    control.qianzhan = CarParams->qianzhan_low + 
      static_cast<int>(static_cast<float>(CarParams->speedHigh - control.motorSpeed) / 
      static_cast<float>(CarParams->speedHigh - CarParams->speedLow)) * 
      (CarParams->qianzhan_high - CarParams->qianzhan_low);
  }
  
  // 串口发送控制指令
  uart->carControl(control.motorSpeed, control.servoPwm);
}

6. 总结

本项目实现了基于EdgeBoard的智能车控制系统，完成了以下工作：

  1. 硬件集成：完成了EdgeBoard与传感器、执行器的硬件连接与调试

  4. 运动控制：实现了PID控制与动态前瞻相结合的路径跟踪算法
  5. 元素处理：完成了圆环、十字、坡道、充电区等多种元素的识别与处理

该项目获得了第20届全国大学生智能汽车竞赛完全模型组全国一等奖。
