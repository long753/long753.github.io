1.打开小车

首先，观察小车的上位机，主控，驱动板的开关是否处于关闭状态（分别位于小车的左右两边）。其次，把电源插上，注意观察电池的电压，要在11.6v到12.6v之间，不要过充过放。最后，打开那三个开关，等待edgeboard上的指示灯显示为绿色，小车成功启动。  

2.连接电脑

2.1.1 有线连接（第一次连接优先这种方式）

  1. 用网线连接电脑和智能小车
  2. 打开终端，输入指令：（有可能连接不上，多试几次）
1

ssh edgeboard@paddlepi

然后输入密码：edgeboard，即可连接成功。

2.1.2 有线连接(路由器连接方式)

  1. 用网线连接路由器和智能小车，同时电脑也连接这个路由器，进入路由器的管理界面，找到智能小车的IP地址。
  2. 在电脑上打开终端，输入指令：
1

ssh edgeboard@paddlepi   #换成对应的ip地址
  1. 输入edgeboard的密码：edgeboard，即可连接成功。

2.2 无线连接（最常用）

第一次无线连接时：保证有线连接成功，然后在终端中输入

1

sudo nmtui

然后点击active connect连接你的热点即可。

如果不是第一次连接，只需打开手机热点让算力版连接到你的热点即可。

打开终端，输入指令：（有可能连接不上，多试几次）

1

ssh edgeboard@paddlepi

密码：edgeboard

3.连接vscode

3.1安装插件

首先在vscode里面安装Remote - SSH，这一步很ez

3.2.在插件里面点进car-work

  1. 点击ssh-remote的“+”号
  2. 在输入栏中输入：
1
2

ssh edgeboard@paddlepi
密码：edgeboard
  1. 一直允许，配置文件点第一个就可以了（记住这个配置文件的位置）
  2. 在vscode里面打开car-work文件夹
1

/home/edgeboard/car-work

注意：这里很有可能会连接失败，经过我的多次测试，我发现有两个可能，第一个是你的vscode版本太高，你去下载一个相对老一点版本的VScode就可以了，第二种情况可以通过下面的方式的解决： ![[Pasted image 20250429000335.png]] 点击上面图片中的扩展设置，将下面的这个路径改成你当时用ssh连接时的算力版的配置文件，忘记了也没事，你可以通过ssh中的查看配置来找到这个文件，而且大概率也跟我的路径差不多 ![[Pasted image 20250429000429.png]] 之后你只需要点击并输入密码即可连接

4.打开VNC图形化界面

如果你还没有安装VNC，请打开网址https://www.realvnc.com/en/connect/download/viewer/，安装windows版本的VNC客户端，并按照安装提示安装好。

进入edgeboard终端后，输入指令：

1

vncserver :1 -localhost no

其中：:1表示图形化界面端口号，可以自行设置。优先使用1号端口，当端口被占用时，可以尝试其他端口,只使用1~4号端口。

当出现下面的提示则说明启动成功：

1

vnc vnc vnc vnc

然后打开VNC,建立新的连接：（如果连接过，直接点你原来建立的就ok）

1
2

vncserver：paddlepi:5901
name: edgeboard

同时，如果使用的不是1端口，例如是2端口，对应的应该为：vncserver：paddlepi:5902 进入后输入密码：edgeboard

#修复环境
sudo apt --fix-broken install -y
sudo apt update
sudo apt install -y build-essential xz-utils bison flex wget

#拉取源码
cd /home/edgeboard/Download
wget https://mirrors.nju.edu.cn/gnu/gcc/gcc-11.2.0/gcc-11.2.0.tar.xz
sha256sum gcc-11.2.0.tar.xz
tar -xf gcc-11.2.0.tar.xz

#拉取依赖并编译
cd gcc-11.2.0
./contrib/download_prerequisites
cd ..
mkdir -p gcc-11.2.0-build && cd gcc-11.2.0-build
../gcc-11.2.0/configure \
  --prefix=/opt/gcc-11.2 \
  --enable-languages=c,c++ \
  --disable-multilib \
  --enable-bootstrap
make -j"$(nproc)"  #需要很长一段时间，大概一坤时，请准备预留好时间，后续将很快
sudo make install

#修改PATH
echo 'export PATH=/opt/gcc-11.2/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/opt/gcc-11.2/lib64:${LD_LIBRARY_PATH}' >> ~/.bashrc
source ~/.bashrc

#验证
which gcc
gcc --version
g++ --version

5.2 libavdevice-dev 

1

sudo apt-get install libavdevice-dev

6.编译文件

在VNC界面中，打开终端，切换到工程目录，输入指令：

1
2
3
4
5

 cd /home/edgeboard/car-work  #切换到自己的那个工作空间里面
 rm -rf build
 mkdir build
 cmake ..   #如果你修改了整个代码框架，可能需要修改一下CMakeList文件，如何修改可以去问AI
 make icar -j3

完全配置完全之后，保存你修改的代码后，进入build目录输入指令即可完成编译：

1

 make icar -j3

其中，“icar”表示编译的目标文件，“-j3”表示使用三个线程编译，可以加快编译时间。这里的编译时间会根据你的修改程度而变化。

7.运行程序

在终端中，切换到工程目录，输入指令：

1

 ./icar

此时小车便成功启动，结束小车的运动只需在终端中输入指令Ctrl+C即可。
