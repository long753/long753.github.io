1. 项目概述

本项目设计了一款6-DOF全驱动水下机器人（AUV），旨在完成水下抓取任务与高性能运动控制。与传统欠驱动或冗余驱动的AUV相比，6-DOF全驱动方案能够在紧凑设计的基础上实现更高水平的运动控制能力，完成复杂的水下作业任务。此外，为同时满足高精度水下抓取与高性能运动控制，本文设计了AUV的两种运动模式：抓取模式与巡航模式。抓取模式使用传统控制方案以提升其精度，巡航模式使用DRL驱动以实现其复杂的运动解耦。

1.1 设计目标

  • 高性能的机械结构设计
  • 传统控制方案设计
  • DRL控制方案设计
  • 目标识别与抓取
  • 自主路径导航

1.2 技术支持

  • mjlab & mojoco
  • ROS2
  • SSH & Git
  • python & C++

2. 结构设计

2.1 推进器分布方案

传统推进器分布分为三类：欠驱动、全驱动和过余驱动。

类型特点适用场景
欠驱动推进器数量少于自由度数简单观测任务
全驱动推进器数量等于自由度数精细作业任务
过余驱动推进器数量多于自由度数高冗余任务

考虑到机器人需要在紧凑设计的基础上实现高水平的抓取任务，本设计选用6-DOF全驱动方案。

常规全驱动方案使用6个固定推进器或4个矢量推进器。矢量推进器虽然机动性好，但结构复杂、可靠性较低。因此，本设计提出了一种4推进器+2舵机的混合推进方案：

  • 4个主推进器提供主要推力
  • 2个舵机控制机械臂角度

  • 浮力材料：EPP泡沫

3. 控制方案

3.1 运动学建模

建立机器人坐标系，使用DH参数法进行运动学建模：

关节$\theta$$d$$a$$\alpha$
1$\theta_1$$d_1$0$90°$
2$\theta_2$0$a_2$0
3$\theta_3$0$a_3$0
4$\theta_4$00$90°$
5$\theta_5$00$-90°$
6$\theta_6$$d_6$00

正运动学方程：

$$T_i^{i-1} = \begin{pmatrix} \cos\theta_i & -\sin\theta_i\cos\alpha_i & \sin\theta_i\sin\alpha_i & a_i\cos\theta_i \ \sin\theta_i & \cos\theta_i\cos\alpha_i & -\cos\theta_i\sin\alpha_i & a_i\sin\theta_i \ 0 & \sin\alpha_i & \cos\alpha_i & d_i \ 0 & 0 & 0 & 1 \end{pmatrix}$$

3.2 传统控制方案

对于抓取模式，采用PID控制器进行底层运动控制：

class PIDController {
public:
    double kp, ki, kd;
    double setpoint, current_value;
    double integral = 0;
    double last_error = 0;
    
    double compute(double feedback) {
        double error = setpoint - feedback;
        integral += error;
        double derivative = error - last_error;
        last_error = error;
        return kp * error + ki * integral + kd * derivative;
    }
};

PID控制器参数整定：

  • 位置控制：$K_p = 1.5$, $K_i = 0.2$, $K_d = 0.5$
  • 姿态控制：$K_p = 2.0$, $K_i = 0.3$, $K_d = 0.8$

3.3.1 场景配置（Scene）

  • 环境数量: 支持大规模并行训练
  • 地形类型: 平地（plane）水下环境
  • 机器人: AUV 无人水下航行器

3.3.2 动作空间（Actions）

AUV 具有两类执行器：

执行器类型关节名称作用动作尺度
舵机joint_servo_up, joint_servo_left控制方向舵角度±1.5 rad
推进器joint_prop_up, joint_prop_down, joint_prop_left, joint_prop_right提供推进力±200 N

3.3.3 指令空间（Commands）

速度指令生成器使用 UniformVelocityCommandCfg，生成的指令包括：

  • 线速度指令:
    • X轴（前后）: -1.0 ~ 2.0 m/s
    • Y轴（侧向）: -1.0 ~ 1.0 m/s
  • 角速度指令:
    • Z轴（偏航）: -1.0 ~ 1.0 rad/s
  • 指令重采样时间: 3~8秒

3.3.4 观测空间（Observations）

Agent 接收的观测信息包括：

观测项描述维度
command目标速度指令4
base_lin_vel机身线速度3
base_ang_vel机身角速度3
projected_gravity重力投影向量3
joint_pos舵机角度2
joint_vel关节速度6
last_action上一帧动作8

3.3.5 奖励函数（Rewards）

奖励设计分为核心目标奖励和惩罚项：

核心奖励:

  • track_linear_velocity: 跟踪目标线速度（权重 1.0）
  • track_angular_velocity: 跟踪目标角速度（权重 1.0）

惩罚项:

  • body_orientation_l2: 惩罚非水平姿态，保持平稳（权重 -1.0）
  • body_ang_vel: 惩罚 Roll/Pitch 轴角速度，防止晃动（权重 -0.05）
  • action_rate_l2: 惩罚动作变化率，防止电机抖动（权重 -0.01）
  • action_l2: 惩罚动作幅值，鼓励节能（权重 -0.005）
  • dof_vel_z: 惩罚 Z 轴线速度，防止上下窜动（权重 -0.05）

3.3.6 事件系统（Events）

  • 关节重置: 训练开始时重置机器人关节状态
  • 水流扰动: 每隔 5-10 秒随机施加外部推力，增强鲁棒性

3.3.7 终止条件（Terminations）

  • time_out: episode 时间超时（默认 20 秒）
  • fell_over: 机身姿态超过 70° 倾斜

6. 总结

本文设计了一款6-DOF全驱动水下机器人，主要贡献包括：

  1. 推进方案创新：提出4推进器+2舵机的混合推进方案
  2. 控制策略优化：引入深度强化学习实现自主抓取
  3. 紧凑结构设计：在有限空间内实现全驱动配置

该机器人可应用于水下探测、文物打捞、海洋生物研究等领域。

6.1 改进方向

  • 进一步优化机械臂末端精度
  • 增强视觉算法在浑水环境下的鲁棒性
  • 延长水下续航时间
