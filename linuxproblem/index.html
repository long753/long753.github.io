二、常见错误

1. 显卡驱动可能导致的问题：

出现下列问题都先配置一下显卡驱动

  • 屏幕闪烁
  • 双屏分配率问题

2. 屏幕无法调节亮度

使用xrandr来强制调节屏幕亮度

# 1. 确认你的显示器代号
xrandr | grep " connected" # 假设输出开头是DP-4

# 2. 调节亮度到 0.6（范围 0.0-1.0） 
xrandr --output DP-4 --brightness 0.6

3. WiFi图标消失

WiFi 图标自动消失的原因是 Ubuntu 在执行 sudo apt update && sudo apt upgrade 时，默认启用了内核自动升级服务（unattended-upgrades），导致内核从 5.15.0-67-generic 升级至 5.15.0-130-generic。由于新内核与 AX211 网卡驱动不兼容，驱动加载失败，固件文件无法正确加载，且内核模块版本不匹配，最终导致无线网卡无法正常工作，WiFi 图标消失。 可以使用 uname -r 命令检查当前内核版本，如果与 WiFi 正常工作时的内核版本不一致，则很可能是系统自动更新后，内核升级导致新版本与 AX211 网卡驱动不兼容，从而影响 WiFi 功能。

  1. 回滚到旧内核版本
    重启后，在引导界面选择 “Ubuntu 的高级选项” 进入 GRUB 菜单，然后选择原来的内核版本 (5.15.0-67-generic) 进行启动，并测试 WiFi 功能是否恢复正常。
  2. 锁定当前内核版本
1
2
3
4

sudo apt-mark hold linux-image-5.15.0-67-generic
sudo apt-mark hold linux-headers-5.15.0-67-generic
sudo apt-mark hold linux-modules-5.15.0-67-generic
sudo apt-mark hold linux-modules-extra-5.15.0-67-generic
  1. 验证 输入apt-mark showhold查看是否锁定成功
  2. 修改GRUB配置 sudo nano /etc/default/grub

4. 双击图标缩小

配置：gsettings set org.gnome.shell.extensions.dash-to-dock click-action 'minimize' 复原：gsettings reset org.gnome.shell.extensions.dash-to-dock click-action
